## Project Overview

Green-Cycle simulates a **smart city waste assistant**. Citizens describe a waste item in plain text and receive:

1. A **waste category** — `Recyclable`, `Compost`, or `Hazardous`
2. A **human-readable disposal plan** generated by an LLM based on simulated city policy

The system is built as a production-style Python application combining ML engineering, LLM agent design, REST API development, and Docker containerisation.

---

## System Workflow

```
User Text Input
      │
      ▼
FastAPI Endpoint (/classify or /disposal)
      │
      ▼
NLP Preprocessing (spaCy lemmatisation + stopword removal)
      │
      ▼
TF-IDF Vectorisation
      │
      ▼
Logistic Regression Classifier
      │
      ▼
Predicted Waste Category
      │
      ▼
AI Agent — City Policy Lookup (dictionary)
      │
      ▼
Few-Shot Prompt Construction
      │
      ▼
LLM API Call (Groq / OpenAI-compatible)
      │
      ▼
JSON Response (category + disposal plan)
```

## Setup Instructions

### Prerequisites

- Python 3.10+
- Docker (for containerised deployment)
- A free [Groq API key](https://console.groq.com) — or any OpenAI-compatible LLM provider

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/green-cycle.git
cd green-cycle
```

### 2. Create a Virtual Environment

```bash
python -m venv venv
source venv/bin/activate        # macOS / Linux
venv\Scripts\activate           # Windows
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Download spaCy Language Model

```bash
python -m spacy download en_core_web_sm
```

## Environment Variables

Create a `.env` file in the project root by copying the example:

```bash
cp .env.example .env
```

Edit `.env` and fill in your values:

```env
# LLM Provider Configuration
LLM_API_KEY=your_api_key_here
LLM_MODEL=llama-3.1-8b-instant
LLM_BASE_URL=https://api.groq.com/openai/v1/chat/completions
LLM_TIMEOUT=15
LLM_TEMPERATURE=0.3
```

> **Note:** The system uses an OpenAI-compatible API interface. To switch providers, update `LLM_BASE_URL` and `LLM_MODEL` only — no code changes required.

> **Security:** Never commit `.env` to version control. It is listed in `.gitignore`.


```bash
docker run -p 8000:8000 \
  -e LLM_API_KEY=your_api_key_here \
  -e LLM_MODEL=llama-3.1-8b-instant \
  -e LLM_BASE_URL=https://api.groq.com/openai/v1/chat/completions \
  green-cycle
```

API will be available at:

```
http://localhost:8000
```

> **Note:** API keys are injected at runtime using `-e` flags and are never stored inside the Docker image.

## API Endpoints

### `POST /classify`

Classifies a waste item description and returns the predicted category.

**Request body:**
```json
{
  "text": "empty plastic water bottle"
}
```

**Response:**
```json
{
  "category": "Recyclable",
  "confidence": 0.9341
}
```

---

### `POST /disposal`

Classifies the waste item and returns a full LLM-generated disposal plan.

**Request body:**
```json
{
  "text": "used AA batteries"
}
```

**Response:**
```json
{
  "category": "Hazardous",
  "confidence": 0.9712,
  "plan": "These batteries are classified as hazardous waste. According to city policy, batteries must not be placed in general waste bins. Please seal them in a zip-lock bag and drop them off at the Hazardous Waste Collection Facility on Main Street, open Monday to Saturday 8am–5pm."
}
```

---

### HTTP Status Codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `400` | Bad request — empty or invalid input text |
| `500` | Internal server error — model or LLM failure |

## ML Evaluation Summary

| Metric | Value |
|--------|-------|
| Dataset size | 301 samples |
| Classes | Recyclable, Compost, Hazardous |
| Train / Test split | 80 / 20 stratified |
| Model | Logistic Regression (L2, C=0.5) |
| Features | TF-IDF unigrams (lemmatised) |
| Train Accuracy | ~0.95 |
| Test Accuracy | ~0.92 |
| CV Mean (5-fold) | ~0.91 |
| CV Std | ~0.02 ✅ Stable |
| Train–Test Gap | ~0.03 ✅ No overfitting |

Full evaluation including confusion matrix, learning curves, and C-value comparison is in `notebooks/green_cycle_ml.ipynb`.

## Design Decisions

**Why Logistic Regression?**
TF-IDF produces high-dimensional sparse vectors. Logistic Regression is purpose-built for this — L2 regularisation controls overfitting, it outputs class probabilities for confidence scoring, and it is fast enough for real-time API inference.

**Why a single joblib pipeline?**
Bundling TF-IDF and the classifier into one `sklearn.Pipeline` guarantees that the exact same `clean_text` preprocessing used during training is applied at inference — eliminating train/serve skew.

**Why Groq as the LLM provider?**
Groq offers a free tier with fast inference on open-source models (LLaMA 3). The system uses an OpenAI-compatible interface, so switching to OpenAI, Together AI, or any other provider requires only a `.env` change.

**Why a policy dictionary instead of a real API?**
The assessment requires demonstrating agent logic — the dictionary lookup is the deliberate decision step before the LLM is called. This pattern can be replaced with a real municipal API without changing the agent's decision structure.
